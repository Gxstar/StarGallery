这是一份为您精心整理的**《现代 Android 本地图库开发必备插件/库清单》**。

这份清单专注于 **Kotlin + XML** 技术栈，旨在帮助您从零构建一个具备**工业级性能**（加载万张图片不卡顿）、**现代架构**（解耦清晰）且符合 **Android 14 规范**的图库应用。

| 类别 | 插件 / 库名 | 主要功能 | 为什么它是必备的？ |
| :--- | :--- | :--- | :--- |
| **图片加载核心** | **Glide** | 负责缩略图加载、内存缓存、磁盘缓存及 GIF 播放。 | 图库的心脏。在处理 MediaStore 缩略图和 Bitmap 回收方面，它是 XML 架构下的行业标准。 |
| **大图查看引擎** | **SubsamplingScaleImageView** | 专门处理 4K/8K 超高清图片和长图的缩放查看。 | **不可替代**。原生 ImageView 无法显示高清原图（会模糊或 OOM），它通过分块渲染（Tiling）解决此问题。 |
| **数据流控 (分页)** | **Paging 3** | 实现从系统 MediaStore 数据库到 UI 的分页懒加载。 | **性能关键**。当手机有 10,000+ 照片时，它确保每次只读取几十张，防止 App 启动时卡死或内存崩溃。 |
| **异步架构** | **Coroutines & Flow** | 负责所有的后台任务（读库、解析 EXIF、修改文件）。 | 替代过时的 Thread 或 RxJava，提供极其优雅的非阻塞式代码编写方式。 |
| **依赖注入** | **Hilt (Dagger)** | 自动化管理各个组件（如 Repository、ViewModel）的生命周期。 | 现代架构标配。让代码解耦，便于后期维护，不再需要手动传递各种 Context 对象。 |
| **页面导航** | **Navigation Component** | 管理 Fragment 之间的跳转逻辑和大图预览时的“共享元素动画”。 | 简化 XML 架构下的页面路由，能轻松实现像 Google Photos 那样丝滑的打开照片动画。 |
| **UI 列表分组** | **Groupie** | 快速实现“按日期分组”的多类型 RecyclerView 布局。 | 图库通常需要按天、按月显示标题，Groupie 让复杂的列表代码（Header + Grid）变得异常简单。 |
| **系统 API 交互** | **MediaStore API** | 调用系统原生 API 获取媒体列表及实现**“收藏”**功能。 | **核心需求**。通过 `createFavoriteRequest` 即可实现与系统相册同步的收藏功能，无需自建数据库。 |
| **权限适配** | **PermissionX** | 适配 Android 11 至 Android 14 极其复杂的存储权限请求。 | 郭霖维护。自动处理“部分照片访问权限”等边缘情况，避免因权限问题导致 App 闪退。 |
| **元数据解析** | **ExifInterface** | 读取和写入照片的 EXIF 信息（ISO、焦距、经纬度、拍摄时间）。 | 用户查看照片详情（i 按钮）时的所有数据都源自这个官方支持库。 |
| **视频播放预览** | **Media3 (ExoPlayer)** | 负责图库内视频文件的流畅预览和播放。 | Google 官方的多媒体库，支持几乎所有视频编码格式，且性能极佳。 |
| **快速滚动条** | **RecyclerView-FastScroll** | 为长列表提供带时间悬浮气泡（例如显示“2023年10月”）的滚动条。 | 必备交互。当照片极多时，没有快速滚动条用户会划到绝望。 |
| **内存监控** | **LeakCanary** | 自动检测 Activity、Fragment 或大图对象是否发生了内存泄漏。 | **调试必备**。图库处理 Bitmap 极易泄露内存，该库能在开发阶段帮你找出所有“吃内存”的元凶。 |

---

### 开发核心思路总结：

1.  **加载逻辑**：使用 `Paging 3` 配合 `MediaStore` 进行分段读取，确保启动速度。
2.  **展示逻辑**：
    *   **网格页**：用 `Glide` 加载 MediaStore 的 `Thumbnail`（不要加载原图）。
    *   **详情页**：用 `SubsamplingScaleImageView` 加载原图 URI，保证缩放细节。
3.  **收藏逻辑**：不要在本地建 Room 数据库存收藏状态，直接调用 `MediaStore.createFavoriteRequest`。这样用户在你的 App 里点收藏，系统相册里也会同步显示为收藏。
4.  **性能优化**：通过 `LeakCanary` 严格监控每一个 Fragment 销毁后的内存释放，图库 App 的成功全在于**内存管理**。
